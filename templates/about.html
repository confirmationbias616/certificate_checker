HBR Bot assists Ontario subcontractors in quickly recovering their 10% holdback funds. This is acheived by monitoring all known publishers of CSP's (Certificates of Substantial Completion) and notifying involved subcontractors once a match has been found for one of their projects. Upon receiving an alert, subcontractors can then promptly invoice their holdback release with confidence.


Why start using this tool now?    

Before 2019, Daily Commercial News (DCN) used to have a monopoly on publishing of CSP's in Ontario. Due to a recent change in provincial legisltion, the market has been opened up to allow competition in CSP publishing. As shown above, the proliferation of publishers has already begun.

Before long, it will become far too time consuming for anyone to check CSP sources regularly and cross-reference with their own list of open projects. HBR Bot streamlines this process and makes it available for all.

To get started, simply enter a new project and let HBR Bot guide you through the process. To learn more about the Lien Construction Act and holdback releases in general, visit this link.


Tech Stack

HBR Bot is written in Python and SQL. In essence, it is a machine learning model wrapped in a web app. For intelligent matching of CSP's to subcontractor's logged open projects, the type of data science problem being solved here can be classified as Entity Resoultion. The machine learning model is a Scikit-Learn Random Forest and it is fed by an active style of learning with human-in-the-loop validation via user feedback from the web app.

Due to the class imbalance issue of an initially small amount of confirmed positive matches, the ML training process makes use of the SMOTE algorithm. The data wrangling step utilizes the fuzzywuzzy package, which applies fuzzy logic to all string mtaching.

The web app is built with Flask and is hosted on PythonAnywhere.

The data is stored in SQLite3 and the FTS3 extension is used for text search within the database.

The graphs are built with Matplotlib and the maps are built with Folium / Leaflet / Open Street Map.

Project data is enriched with the Google Maps Geocode API.

Authentication and authorization of users is provided via Google's Sign-In OAuth API.

Rate limiting is managed between web workers by a Redis key-value store via Redislite.

Payments are processed with Stripe and emails are sent with Sendgrid.

The code base is version controlled with Git and integrated with GitHub.It is also automatically tested and continuously integrated on CircleCI.
